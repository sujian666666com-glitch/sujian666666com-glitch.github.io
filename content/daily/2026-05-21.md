## 1. The SpaceX IPO filing: AI bets, Starship dreams, Elon Musk

- 标题：The SpaceX IPO filing: AI bets, Starship dreams, Elon Musk
- 摘要：SpaceX 公开提交 IPO 注册文件，拟以 `SPCX` 在 Nasdaq 上市。TechCrunch 报道称，文件和相关披露把 Starlink、Starship、xAI/Grok、AI 基础设施和数据中心等纳入资本市场叙事；其中 xAI 相关业务在 2025 年产生高额资金消耗。更准确的表述是：文件与报道显示 AI 已成为 SpaceX IPO 增长叙事中的重要部分，而不是说 SpaceX 已经从航天公司转型为 AI 公司。

## 2. Cohere releases Command A+, a new open-source model focused on enterprise AI

- 标题：Cohere releases Command A+, a new open-source model focused on enterprise AI
- 摘要：Cohere 发布 Command A+，并以 Apache 2.0 许可开放权重。Cohere 称该模型是其目前最快、最强的语言模型，面向企业级 agentic workflow、复杂推理、多语言任务和私有化部署；模型卡显示其为 MoE 架构，218B 总参数、25B 激活参数，支持 128K 输入上下文、48 种语言、工具调用和图像输入。Hugging Face 页面还说明，W4A4 量化版本的最低示例配置为 2 张 H100 或 1 张 B200，因此硬件门槛应限定在量化版本语境下描述。
- 链接：[Cohere 官方公告](https://cohere.com/blog/command-a-plus) / [Hugging Face BF16 模型卡](https://huggingface.co/CohereLabs/command-a-plus-05-2026-bf16) / [Hugging Face W4A4 模型卡](https://huggingface.co/CohereLabs/command-a-plus-05-2026-w4a4) / [TechCrunch 报道](https://techcrunch.com/2026/05/21/cohere-releases-command-a-a-new-open-source-model-focused-on-enterprise-ai/)

## 3. Meet Stable Audio 3.0: The model family built for artistic experimentation with open-weight models

- 标题：Meet Stable Audio 3.0: The model family built for artistic experimentation with open-weight models
- 摘要：Stability AI 发布 Stable Audio 3.0 模型家族，共包含 Small SFX、Small、Medium 和 Large 四款模型。官方称该系列使用完全授权数据训练，其中 Small SFX、Small 和 Medium 为开放权重模型，可在 Hugging Face 获取；Large 通过 Stability AI API 和企业自托管提供。新能力包括可变长度音频生成：Small 最高约 2 分钟，Medium 和 Large 可生成超过 6 分钟，Medium 标称最高约 6:20；输出商业化和企业使用还需遵守 Stability AI Community License 或 Enterprise License 条款。
- 链接：[Stability AI 官方公告](https://stability.ai/news-updates/meet-stable-audio-3-the-model-family-built-for-artistic-experimentation-with-open-weight-models) / [TechCrunch 报道](https://techcrunch.com/2026/05/20/stability-ai-launches-new-open-models-for-generating-audio/)

## 4. GitHub internal repos stolen in VS Code extension attack

- 标题：GitHub internal repos stolen in VS Code extension attack
- 摘要：CyberScoop 报道称，GitHub 表示一台员工设备因被投毒的 VS Code 扩展遭到入侵，导致部分 GitHub 内部仓库被外传。GitHub 称已检测并遏制该事件，移除恶意扩展版本、隔离受影响终端，并轮换关键 secrets；攻击者声称约 3,800 个仓库受影响，GitHub 仅称这一说法与当前调查方向一致，完整范围仍在调查中，并表示会在调查完成后发布更完整报告。GitHub 当前说法是该活动限于内部仓库，尚无证据显示受影响仓库之外的客户数据受到影响。
- 链接：[CyberScoop 报道](https://cyberscoop.com/github-internal-repositories-vs-code-extension-attack/) / [Aikido 分析](https://www.aikido.dev/blog/github-breached-vs-code-extension) / [GitHub 公告](https://x.com/github/status/2056949168208552080)
