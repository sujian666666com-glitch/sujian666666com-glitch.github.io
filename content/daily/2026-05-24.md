## 1. Scammers are abusing an internal Microsoft account to send spam links

- 标题：Scammers are abusing an internal Microsoft account to send spam links
- 摘要：TechCrunch 报道称，诈骗者利用 Microsoft 账号通知系统中的漏洞，从 `msonlineservicesteam@microsoftonline.com` 这类常用于账号提醒的官方地址发送垃圾链接邮件。Spamhaus 表示类似滥用已持续数月；Microsoft 在报道更新中称正在调查并加强检测、拦截和违规账号移除。这件事值得关注，因为它削弱了“来自官方通知地址就更可信”的反钓鱼判断。
- 链接：[原始报道](https://techcrunch.com/2026/05/21/scammers-are-abusing-an-internal-microsoft-account-to-send-spam/) / [HN 讨论](https://news.ycombinator.com/item?id=48253186)，热度：约 198 points / 103 comments

## 2. AMD Vivado 2026.1 引发 Basic 免费层 Linux 支持争议

- 标题：Why is Vivado 2026.1 dropping Linux support for free tier?
- 摘要：AMD 的 Vivado 2026.1 新分层授权页面显示，Basic 层仍为免费但 OS Support 表格中 Windows 可用、Linux 未勾选，Core 及以上层级才显示 Linux 支持；AMD 同页说明新分层模型从 2026.1 开始。由于 FPGA 学生、爱好者、小团队和 CI 环境常依赖 Linux，HN 与 Slashdot 讨论集中在免费层平台限制是否会抬高入门门槛。AMD 另有安装器 OS 支持页显示 2026.1 工具本身仍计划支持多种 Linux 发行版，因此更准确的说法是“免费 Basic 授权层不含 Linux 支持”，而不是“Vivado 2026.1 完全不支持 Linux”。
- 链接：[AMD 授权说明](https://www.amd.com/en/products/software/adaptive-socs-and-fpgas/vivado/vivado-licensing-options.html) / [AMD OS 支持页](https://www.amd.com/en/support/adaptive-socs-and-fpgas/installer-info-general.html) / [Slashdot 报道](https://hardware.slashdot.org/story/26/05/23/1917255/amd-xilinx-is-excluding-linux-from-the-free-tier-for-its-fpga-dev-tool) / [HN 讨论](https://news.ycombinator.com/item?id=48254309)，热度：约 240 points / 114 comments

## 3. Silk: Open-source cooperative fiber scheduler

- 标题：Silk: Open-source cooperative fiber scheduler
- 摘要：ClickHouse 开源了 `silk`，项目 README 将其描述为面向 Linux 的协作式 fiber 调度器，包含每 CPU scheduler thread、`io_uring` 集成和拓扑感知 work-stealing。仓库文档覆盖调度循环、上下文切换、异步 I/O、同步原语、性能测试和 GDB 扩展；这类低层运行时项目对数据库、网络服务和高并发 C++ 系统开发者有直接参考价值。
- 链接：[GitHub 仓库](https://github.com/ClickHouse/silk) / [HN 讨论](https://news.ycombinator.com/item?id=48210937)，热度：约 68 points / 7 comments

## 4. Constraint Decay: The Fragility of LLM Agents in Backend Code Generation

- 标题：Constraint Decay: The Fragility of LLM Agents in Backend Code Generation
- 摘要：arXiv 论文研究 LLM agent 在后端代码生成中随结构性约束增加而性能下降的现象，作者称之为 “constraint decay”。论文设置了 80 个 greenfield 生成任务和 20 个功能实现任务，覆盖 8 个 Web 框架，并用端到端行为测试与静态 verifier 评估结果；结论指出，显式、轻量框架上的表现通常优于 convention-heavy 框架。对正在把 coding agent 接入真实仓库的团队来说，重点不是“换个提示词”，而是要把约束、测试、检查点和框架约定工程化。
- 链接：[arXiv 论文](https://arxiv.org/abs/2605.06445) / [EURECOM 页面](https://www.eurecom.fr/en/publication/8745)
