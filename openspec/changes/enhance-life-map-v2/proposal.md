# LifeMap v2：暖色手账式人物关系闯关地图

## 为什么

当前 LifeMap 已具备独立 Next.js 子应用、密码解锁、服务端数据保护、React Flow 地图、筛选、详情、时间轴、图例和 PNG 导出能力，但整体视觉仍偏深色科技知识图谱，人物关系和人生叙事权重不够清晰。

本次变更希望把 LifeMap 重构为一本暖色手账式人生冒险地图：默认以人物关系为主结构，次级展示闯关路线，并把“她”作为贯穿高中到大学分手阶段的重要青春支线呈现。

## 目标

- 将 LifeMap 从深色科技图谱重绘为米白、奶油、羊皮纸质感的暖色手账地图。
- 增加双模式切换：默认“人物关系”，次级“闯关路线”。
- 增加 `person` 节点类型，人物统一使用称谓化表达，不出现真实姓名。
- 点击人物时，高亮 TA 影响过的关卡、Boss、觉醒点和目标，其它内容降透明。
- 将“她”升级为高权重人物节点，并在路线模式中形成完整、连续、明显的“青春支线”。
- 保持现有密码验证、服务端数据保护、部署入口和 React Flow 基础能力不回退。

## 范围 Scope

- 扩展 LifeMap 类型与服务端数据结构：
  - `LifeNodeType` 增加 `person`。
  - 节点增加 `modeRole`、`visualKind`、`chapter`、`relatedPeopleIds`、`storyBeats`、`defeatBy`、`branchId`、`branchLabel`、`branchWeight`。
  - 边增加 `relationLabel`、`lineStyle`、`strength`、`visibleInModes`。
- 用同一份服务端数据支持两个展示模式：
  - 人物关系地图：以“我”为中心，奶奶、她、爸爸、姑姑、弟弟、表姐、表弟、宿舍朋友、现实提醒者、OpenCV 老师 / 培训老师、未来的我围绕分布。
  - 闯关路线地图：从左下到右上串联“小镇起点 → 初中迷失 → 初三醒悟 → 青春支线 → 大学拖延 Boss → 游乐场觉醒 → AI 转职 → 深圳实习 → 终极目标”。
- 青春支线使用 `branchId: "youth-love"`、`branchLabel: "青春支线"`、`branchWeight: "important"`、`relatedPeopleIds: ["her"]`、`lineStyle: "youth-branch"`、`visibleInModes: ["relationship", "route"]`。
- 青春支线至少包含：高中相遇、高一被发现、短暂分开、没有名分但彼此在意、高二初吻、高考后正式在一起、大一异地分手、至今未联系、长期遗憾。
- 视觉改为暖色手账地图：米白 / 奶油 / 羊皮纸底色，暖白纸卡，深棕文字，柔和边框，轻纸张纹理，手绘山丘、树、村庄、云朵等点缀。
- 主题色限定在：`#F7F1E5`、`#F3E8D2`、`#FFF9EC`、`#332A22`、`#7A6A58`、`#D8C5A8`、`#B76E3C`、`#D6A84F`、`#7B9B6F`、`#9A4A3F`、`#8B6F9A`。
- 顶部模式切换改为手账书签；筛选改为纸质标签：全部、家庭、青春、学习、工作、Boss、觉醒、目标。
- 详情面板改为手账纸卡，并按人物、关卡、Boss、觉醒点、青春支线展示不同信息重点。
- 时间轴改为章节书签：童年、小学、初中、高中、大学、实习、未来。
- PNG 导出仅包含暖色手绘地图主体。

## 不做什么 Out of Scope

- 不迁移 Hugo 主站，不改变 `/about/life-map` 页面入口。
- 不改变 `/api/life-map/verify` 和 `/api/life-map/data` 的密码验证、签名 token、服务端保护方式。
- 不把真实履历数据直接 import 到客户端组件或静态 bundle。
- 不引入 Phaser、PixiJS、Canvas 游戏引擎。
- 不做与 LifeMap v2 无关的全站主题、Hugo 布局或聊天组件改造。
- 不引入真实姓名；人物只使用称谓化表达。

## 关键业务场景与期望结果

### 场景一：默认进入人物关系地图

- 用户解锁 LifeMap 后，默认看到“人物关系”模式。
- 地图中心是“我”，人物节点以 NPC 贴纸 / 小头像纸贴形式围绕分布。
- 奶奶和“她”的视觉权重高于普通人物，其中“她”仅次于奶奶和“我”。

### 场景二：点击人物高亮影响范围

- 用户点击“奶奶”后，小镇起点、家庭责任、终极目标等相关节点和关系线被高亮。
- 与奶奶无关的节点和边降低透明度，但不完全消失。
- 详情面板显示 TA 是谁、与我的关系、影响过哪些关卡、留下的情绪。

### 场景三：点击“她”聚焦青春支线

- 用户点击“她”后，高亮“她”节点和完整青春支线。
- 青春支线节点包括高中相遇、高二初吻、高考后正式在一起、大一异地分手、长期遗憾等关键剧情。
- 主线仍保留但降低透明度，让青春支线成为视觉焦点。

### 场景四：切换到闯关路线地图

- 用户点击“闯关路线”书签后，地图布局切换为从左下到右上的主线闯关路径。
- 青春支线从高中阶段分叉出来，使用紫棕色 / 暖粉棕色虚线延伸到大学异地分手和长期遗憾。
- 青春支线明显、连续、有路径感，不能只是一个孤立节点。

### 场景五：查看青春支线节点详情

- 用户点击青春支线节点后，详情面板展示发生了什么、当时的情绪、对我的影响、为什么成为遗憾。
- 详情还需要说明它与“出人头地 / 自我证明 / 另一条人生线”的关系。

### 场景六：Boss 与觉醒点详情

- 用户点击 Boss 后，详情面板展示弱点、代价、击败方式、当前状态。
- 用户点击觉醒点后，详情面板展示前因、触发、后果。

## 风险与约束

- v2 会显著增加节点和边数量，需要重点控制移动端坐标、画布高度、fitView 和导出稳定性。
- 数据解锁后会完整返回客户端；本次只保持现有“解锁后可见全部”的安全模型，不做分级隐私。
- 视觉大改会触及多个组件，应保持改动集中在 `apps/life-map` 与本 change 文档内，避免误伤主站。
- 搜索 `.next/static` 必须确认真实履历内容仍不进入客户端静态 bundle。
