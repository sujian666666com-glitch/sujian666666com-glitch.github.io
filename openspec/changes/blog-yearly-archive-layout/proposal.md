## 为什么

当前 `/posts/` 和普通博客分类页沿用文档化列表布局，信息清晰但不够像“博客索引”。用户给出的参考图更适合个人博客的浏览入口：顶部有强识别度主题横幅，下方按年份快速浏览文章。

## 改动内容

- 将 `/posts/` 改造成年度索引页：顶部橙红几何主题横幅，下方按年份分组展示文章链接。
- 将包含普通博客文章的分类页使用同一年度索引风格，但列表只展示 `posts` 栏目文章。
- 每条文章展示标题、日期和基础 meta，不展示封面大卡片和长摘要。
- 移动端降级为单列年度列表，横幅字号和高度收敛，避免横向溢出。
- 样式使用 `blog-archive-*` 作用域类名，避免影响首页、单篇文章、每日新闻、小纸条、小虾、聊天组件和留言墙。

## 范围

- 影响 `/posts/`、`/posts/` 下普通博客 section 页，以及包含普通博客文章的 `/categories/<term>/` 页面。
- 允许修改 `layouts/_default/list.html` 和 `assets/css/extended/custom.css`。

## 不做什么

- 不改首页。
- 不改单篇文章详情页。
- 不改 daily、gallery、shrimp-diary、chat、message wall 和任何 API。
- 不调整文章 Markdown 源文件结构。

## 影响

- 普通博客入口会更接近年度历史索引，读者能按年份快速扫读。
- 分类 taxonomy 中如果只包含 daily 或 shrimp 文章，仍走原有列表布局，避免非博客栏目被误套新样式。
