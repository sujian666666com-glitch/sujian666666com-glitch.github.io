## 为什么

上一版 `/daily/` 已改成杂志化档案列表，但 daily 正文仍是上方大 hero、正文居中、TOC 折叠块的传统博客阅读方式。用户给出的参考图更接近文档站阅读体验：左侧导航、中间正文、右侧目录，并且 Markdown 渲染更清晰。

## 改动内容

- 为 daily 页面增加文档化三栏布局：左侧最近日期导航、中间内容区、右侧页内目录。
- `/daily/` 列表页保留杂志档案条目，但套入左侧日期导航和右侧辅助目录。
- daily 正文页移除过重的顶部 hero，改为文档式标题区：面包屑、栏目标签、大标题、meta。
- 右侧 TOC 在桌面端固定显示，移动端降级为正文上方折叠目录。
- 增强 daily Markdown 渲染：标题、段落、列表、链接、inline code、blockquote、表格、代码块和“简短结论”。
- 样式继续使用 daily 专属 class，不影响首页、普通博客、小 k、留言墙和其它栏目。

## 范围

- 只影响 `/daily/` 列表页和 `/daily/<slug>/` 正文页。
- 允许新增 daily 专用 partial，以减少列表页和正文页的重复模板。

## 不做什么

- 不改每日新闻 Markdown 源文件结构。
- 不改首页、普通 `/posts/`、相册、小虾日记、小 k、RAG、留言墙或 API。
- 不新增 JS 交互，不引入新前端框架。

## 影响

- daily 页面会更接近文档站：桌面宽屏信息更密，目录始终可见。
- 移动端会取消固定三栏，保证正文优先可读。
