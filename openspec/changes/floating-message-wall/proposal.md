## 为什么

当前 `/gallery/` 是传统相册展示，和博客整体“安静、轻盈、干净”的个人空间气质不够贴合。希望把它改成一个极简匿名留言空间：访客路过时只留下一句话，这句话以毛玻璃卡片漂浮在页面中。

## 改动内容

- 将 `/gallery/` 改为“路过的人”匿名留言墙。
- 前台只展示漂浮留言卡片和底部加号，不展示标题、昵称、时间、头像、点赞、回复和楼层。
- 使用浅色渐变背景、光斑、毛玻璃卡片、柔和阴影和轻微漂浮视觉。
- 留言墙可向下滚动，留言越多页面越向下延伸。
- 访客可通过底部加号打开弹窗并匿名提交 1–80 字纯文本。
- 使用服务器本地 SQLite 保存留言，由 `43.135.48.207` 上的 Node API 服务处理公开接口和管理接口，后台保留提交时间、IP、User-Agent、Referer。
- 提供隐藏管理页用于查看全部留言、隐藏、恢复和删除。
- 全站每天最多提交 100 条留言，按北京时间自然日统计。

## 范围

- 改造 `/gallery/` 页面展示与交互。
- 新增服务器本地 Node API 服务，增加留言墙公开接口和管理接口。
- 使用 SQLite 文件数据库保存 `wall_messages`，服务启动时自动初始化表结构和索引。
- 增加 systemd 服务和 Nginx `/api/wall/` 反代配置说明。
- 增加隐藏后台页 `/wall-admin/`。

## 不做什么

- 不做登录、昵称、头像、点赞、回复、分享、排行榜。
- 不预置初始留言。
- 不继续依赖 Cloudflare D1 作为留言墙数据源。
- 不在前台展示来源、身份、时间和管理信息。

## 影响

- 原 `/gallery/` 相册内容会被留言墙替换。
- 需要在 `43.135.48.207` 安装 Node 运行时并配置 `WALL_ADMIN_TOKEN` 后，线上提交与后台管理才可用。
