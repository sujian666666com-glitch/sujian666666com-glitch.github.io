## 为什么

当前 `/daily/` 每日新闻沿用通用文章列表卡片样式，封面占比偏大，整体更像普通博客流。用户希望参考 `jiangyangcreate.github.io` 的杂志极简气质，把每日新闻呈现为连续档案：强调日期、期数、标题、摘要和阅读入口，弱化封面图，同时保留既有新闻正文格式。

## 改动内容

- 为 `/daily/` 新增专用列表模板，不再复用普通文章大卡片。
- 列表页顶部增加杂志式 masthead，展示栏目名、累计期数、最近更新日期、RSS 和归档入口。
- 每条新闻以档案列表形式展示：日期编号、标题、摘要、小缩略图和阅读入口。
- 封面图保留但弱化为小缩略图；无封面时退化为日期块。
- 为 daily 正文页增加专用头部和内容视觉样式，保留 Markdown 正文和「标题 + 摘要 + 链接」内容契约。
- 样式使用 daily 专属 class，避免影响首页、普通文章列表、小 k、留言墙、相册和小虾日记。

## 范围

- 只影响 `/daily/` 列表页和 `/daily/<slug>/` 正文页。
- 只新增或调整 Hugo 模板、daily 作用域 CSS 和本变更的 OpenSpec 文档。

## 不做什么

- 不改首页、导航、普通 `/posts/` 列表和其它栏目。
- 不迁移技术栈，不引入 Docusaurus 或新的前端框架。
- 不改每日新闻 Markdown 生成格式，不批量迁移历史文章。
- 不改小 k、RAG、留言墙、部署服务和 API 配置。

## 影响

- `/daily/` 会从大卡片信息流变成杂志档案列表。
- 每篇 daily 正文页会有更明显的栏目头部和更清晰的新闻条目阅读样式。
- 历史 daily 文章若缺少封面图，列表页会自动显示日期块，不影响访问。
