# 小k 音乐播放器设计方案

## 需求理解

本次目标是在博客聊天组件中加入“小k 音乐播放器”，让用户可以通过聊天触发歌曲推荐与播放。
整体要求是新增独立音乐代理服务，不影响现有聊天、留言墙和博客静态页面。
核心体验是小k回复带播放标记，前端解析后自动搜索歌曲、加入队列并尝试播放。
首版只做搜索、播放、队列和基础降级，不做歌词、收藏、登录或播放历史持久化。

## 目标

- 在聊天面板中新增音乐按钮和播放器抽屉。
- 新增 `music-api.mjs`，代理自管 `NeteaseCloudMusicApi` 实例。
- 支持用户手动搜索歌曲并播放。
- 支持小k通过特殊标记触发前端自动搜歌和播放。
- 上游不可用、无版权、自动播放失败时不影响聊天功能。

## 范围 Scope

- 新增音乐代理服务：搜索歌曲、获取播放 URL、获取歌曲详情。
- 新增聊天面板内播放器 UI。
- 新增播放队列、当前播放、上下首、音量、进度条。
- 修改小k system prompt，使其在推荐歌曲时输出约定播放标记。
- 修改建议词，增加音乐相关入口。

## 不做 Out of Scope

- 不做用户登录。
- 不做收藏、历史播放列表持久化。
- 不做歌词展示。
- 不做代理整段音频流。
- 不做本地曲库管理。
- 不做 STT/TTS 语音对话。

## 当前行为

- 博客已有聊天组件，由 `static/chat-ui.js` 驱动。
- 聊天接口由 `server/chat-api.mjs` 提供，监听本机 `8788`。
- 留言墙接口由 `server/wall-api.mjs` 提供，已有 systemd 与 nginx 配置模式。
- 小k目前只能文字回复，不能触发前端播放行为。

## 期望行为

- 用户点击聊天 header 的音乐按钮后，聊天面板内展开音乐抽屉。
- 用户可搜索歌曲，点击结果后加入队列并播放。
- 用户说“放点轻音乐”时，小k回复中包含 `{{play:歌名|歌手}}`。
- 前端在 assistant 回复完整结束后解析标记，搜索歌曲，选中最佳匹配，加入队列并尝试播放。
- 若自动播放被浏览器拦截，则展开播放器并提示用户点击播放。
- 若上游不可用或歌曲不可播，则展示温和提示，不影响聊天。

## 整体架构

```text
浏览器博客页面
  │
  ├─ static/chat-ui.js
  │    ├─ 聊天 UI
  │    ├─ 音乐抽屉 UI
  │    └─ 小k回复标记解析
  │
  ├─ /api/chat          -> server/chat-api.mjs        -> 大模型上游
  └─ /api/music/*       -> server/music-api.mjs       -> 自管 NeteaseCloudMusicApi
```

## 后端设计

### 新增服务

- 文件：`server/music-api.mjs`
- 默认监听：`127.0.0.1:8789`
- 上游配置：`MUSIC_API_UPSTREAM`
- CORS 配置：`MUSIC_ALLOWED_ORIGINS`，默认复用聊天/留言墙白名单
- 不存数据库，只做转发、标准化响应、内存缓存和错误降级

### API

| 路由 | 作用 | 上游 |
| --- | --- | --- |
| `GET /health` | 本机健康检查 | 无 |
| `GET /api/music/search?q=关键词&limit=10` | 搜歌 | `/search?keywords=...&limit=...` |
| `GET /api/music/url?id=歌曲id` | 获取播放地址 | `/song/url/v1?id=...&level=standard` |
| `GET /api/music/song?id=歌曲id` | 获取歌曲详情 | `/song/detail?ids=...` |

### 响应结构

搜索响应：

```json
{
  "items": [
    {
      "id": 123,
      "name": "歌曲名",
      "artists": ["歌手"],
      "album": "专辑",
      "picUrl": "https://...",
      "duration": 240000
    }
  ]
}
```

播放地址响应：

```json
{
  "id": 123,
  "url": "https://...",
  "playable": true
}
```

错误响应：

```json
{
  "error": "music_unavailable"
}
```

### 缓存策略

- 搜索结果内存缓存 60 秒。
- 播放 URL 内存缓存 10 分钟。
- 缓存只存在于进程内，不做持久化。
- 上游异常时不缓存错误结果。

## 前端设计

### 入口

在聊天 header 的“清除”按钮左侧新增音乐按钮。
按钮样式复用现有 `.chat-header-btn`，交互状态与现有头部按钮保持一致。

### 面板位置

音乐面板放在 `.chat-header` 和 `.chat-messages` 之间，不放进消息滚动流。
展开时占用聊天面板内部高度，消息区自动让出可滚动空间。
移动端限制最大高度，避免挤压输入框。

### 面板功能

- 搜索歌曲。
- 展示搜索结果。
- 当前播放信息：封面、歌名、歌手、进度、时长。
- 播放/暂停、上一首、下一首。
- 音量控制。
- 播放队列。
- 错误提示：不可用、无版权、自动播放失败。

### 播放状态

- 播放队列仅保存在当前页面内存中。
- 刷新页面后不恢复队列、不自动恢复播放。
- 使用标准 `<audio>` 元素播放 `music-api` 返回的音频 URL。
- `audio.play()` 失败时，保留队列并展示“点击播放”提示。

## 小k 联动机制

### 标记格式

```text
{{play:歌名|歌手}}
```

示例：

```text
给你挑一首安静一点的钢琴曲，放进播放器试试。{{play:Merry Christmas Mr. Lawrence|坂本龙一}}
```

### 解析规则

- 只解析 assistant 回复中的标记。
- 一条 assistant 回复最多触发一次。
- 必须等回复完整结束后再触发，不在流式输出过程中触发。
- 历史消息重新渲染时不自动播放。
- 标记字段需要长度限制和 HTML 转义。
- 前端搜索 `歌名 歌手`，优先匹配歌名精确且歌手包含的结果，失败再使用第一条结果。
- UI 中不直接显示原始标记，替换为“已加入播放器”的轻量提示卡片。

## Prompt 改动

在 `server/system-prompt.txt` 末尾增加：

```text
当读者明确想听音乐、让你推荐歌曲或让你放歌时：
1. 用一句话推荐，并在末尾加播放标记 {{play:歌名|歌手}}。
2. 一条回复最多 1 个播放标记。
3. 歌名和歌手尽量选择知名、容易搜到的正版作品。
4. 不要承诺一定已经播放成功，可以说“我给你挑一首放进播放器试试”。
5. 普通聊音乐知识时不要自动加播放标记。
```

## 关键业务场景

### 场景一：手动搜索并播放

- **WHEN** 用户打开聊天面板并点击音乐按钮
- **AND** 在音乐搜索框输入歌曲关键词
- **THEN** 系统展示搜索结果
- **WHEN** 用户点击某首歌曲
- **THEN** 歌曲加入队列并开始播放或提示点击播放

### 场景二：聊天触发自动播放

- **WHEN** 用户发送“放点适合看博客的轻音乐”
- **AND** 小k回复中包含 `{{play:歌名|歌手}}`
- **THEN** 前端在回复完成后解析标记
- **AND** 调用 `/api/music/search` 查找歌曲
- **AND** 调用 `/api/music/url` 获取播放地址
- **AND** 将歌曲加入播放器队列并尝试播放

### 场景三：音乐不可用但聊天可用

- **WHEN** 音乐上游不可达、歌曲无版权或播放 URL 为空
- **THEN** 音乐面板展示不可用提示
- **AND** 当前聊天请求和聊天历史不受影响

### 场景四：历史消息不重复触发

- **WHEN** 用户刷新页面或重新打开聊天面板
- **AND** 历史 assistant 消息中曾出现播放标记
- **THEN** 系统只渲染提示卡片
- **AND** 不再次自动搜索或播放

## 验收标准

- 打开聊天面板后，可以看到音乐按钮。
- 点击音乐按钮，音乐抽屉展开/收起正常。
- 搜索歌曲后，能展示结果并点击播放。
- 小k回复包含 `{{play:歌名|歌手}}` 时，前端能自动搜索并加入队列。
- 自动播放失败时，播放器展示“点击播放”提示。
- 上游不可用时，音乐区域提示不可用，聊天仍可正常使用。
- 刷新页面后，不恢复播放队列。
- 历史聊天重新渲染时，不重复触发播放。

## 风险与降级

- 网易云上游不可用：返回 `music_unavailable`，前端提示音乐暂时不可用。
- 歌曲无版权或 URL 为空：跳过当前歌曲，提示不可播放。
- 搜索错配：通过“歌名 + 歌手”匹配规则降低概率。
- 浏览器自动播放限制：捕获 `audio.play()` 失败，改为用户点击播放。
- 音频 URL 跨域或 mixed content：首版不代理音频流，只做播放失败降级。
